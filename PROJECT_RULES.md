# Инструкция и Архитектура проекта X7 Invest Syndicate Portal

## 🎯 Цель проекта
Создание платформы («Личный кабинет инвестора») для партнеров X7 Invest Syndicate. 
Платформа позволяет инвесторам видеть все свои участия в проектах коммерческой недвижимости в одном месте (статусы, выплаты, прогнозная доходность, документы) и дает ощущение контроля. А также включает в себя Админ-панель для инвест-комитета (создание сделок, управление пользователями и загрузка отчетов).

## 🏗 Архитектура (MVP)
*   **Фронтенд:** React, Vite, Tailwind CSS, Lucide React, Framer Motion. 
*   **Маршрутизация:** React Router (`react-router-dom`). Разделение на две основные зоны:
    *   `/` — Клиентский портал (портфель инвестора, новые проекты, архив документов).
    *   `/admin` — Панель управления (заведение сделок, обновление статусов).
*   **Бэкенд и База Данных (в планах для MVP):** Supabase (PostgreSQL, Auth) или Airtable. Пока используется мок-дата (локальный стейт), но структура компонентов уже готова под подключение реального API.

## 📜 Главные правила разработки

1.  **Визуал — это критически важно.** Сохраняем премиальный, современный и динамичный стиль (в стиле уже созданного макета). Обязательно использование Tailwind CSS для стилизации и Framer Motion для плавных анимаций.
2.  **Структурирование кода.** Больше не пишем всё в одном файле `App.tsx`. Проект разделен на логические страницы (в папке `src/pages`) и компоненты (в будущем).
3.  **Безопасность.** Все административные экраны (`/admin`) в будущем должны быть закрыты авторизацией и проверкой прав доступа.

## 🚀 ПРАВИЛО ДЕПЛОЯ (ВНИМАНИЕ!)

**Любые значимые изменения в логике или визуале на данный момент работают только ЛОКАЛЬНО. Автоматически они никуда не отправляются.**

Проект развёрнут на собственном **VPS (Beget)** — PocketBase сам раздаёт собранный
SPA + API по адресу **https://syndicate-invest.ru**. Vercel больше не используется
(проект ушёл с мок-даты и serverless-хостинга на самостоятельный бэкенд).

### Как деплоить:
1. Закоммитить изменения и `git push` в `origin/main`.
2. На сервере: `cd /root/app && bash deploy.sh`
   (скрипт делает `git reset --hard`, `npm ci`, `npm run build`, копирует `dist`
   в `pb_public`, синхронит миграции PocketBase и перезапускает сервис).
