# Инструкция и архитектура проекта «Личный инвест-портфель»

## 🎯 Цель проекта
Удобный **личный инвест-портфель собственных сделок** в коммерческой недвижимости:
один владелец видит все свои объекты в одном месте — статусы, параметры, фактические
выплаты, прогнозную доходность, документы и фотографии — и держит руку на пульсе.

> Важно: это **не** синдикат-маркетплейс и **не** кабинет для внешних инвесторов.
> Ранние документы (см. `ANALYTIC_NOTE_REAL_ESTATE_MODEL.md`) описывали синдикат с
> инвест-комитетом и множеством инвесторов на сделку — от этой модели проект ушёл.
> Приложение работает в режиме «одна сущность»: вошедший пользователь — владелец
> портфеля с полным доступом (ролевого разграничения нет).

## 🏗 Архитектура
* **Фронтенд:** React + Vite + TypeScript, Tailwind CSS, Lucide React, Motion (Framer Motion).
* **Бэкенд + БД + хостинг:** PocketBase на собственном VPS (Beget). PocketBase сам
  раздаёт собранный SPA из `pb_public` и предоставляет API/Auth.
* **Маршрутизация:** React Router. Единое сквозное меню (сайдбар), все разделы — под
  общим layout (`AppLayout`) за гвардом авторизации (`RoleGuard`):
  * `/` — Главная (обзор портфеля: сводка, распределение активов, календарь выплат, лента событий, карточки сделок).
  * `/analytics` — Аналитика (доходность по объектам, структура, окупаемость, возврат капитала, география).
  * `/payments` — Выплаты (реестр фактических выплат).
  * `/deals` — Сделки (реестр/управление), `/deals/create`, `/deals/:id` (карточка), `/deals/:id/edit`.

### Ключевые модули
* `src/context/DealContext.tsx` — загрузка/сохранение сделок, выплат и файлов; пересчёт метрик.
* `src/utils/mathCore.ts` — финансовые расчёты сделки (NOI, cash flow, cap rate, ROE, DSCR, окупаемость).
* `src/utils/dealMetrics.ts` — хелперы поверх сделки (капитал, годовой поток, окупаемость).
* `src/utils/format.ts` — единое форматирование чисел и денег.
* `src/utils/dealDisplay.ts` — статусы, цвета, прогресс стадий.
* `src/pages/portal/` — страницы и хелперы портала. `src/pages/admin/` — список и форма сделки.

## 📜 Главные правила разработки
1. **Визуал важен.** Современный, аккуратный стиль; Tailwind для стилизации, Motion для анимаций.
2. **Структурирование кода.** Никакого «всё в одном файле»: страницы — в `src/pages`, переиспользуемое — в `src/components` и `src/utils`. Крупные страницы разбиты на модули.
3. **Честность метрик.** Убытки и отрицательная доходность показываются явно (красным), плановые цифры не маскируют факт.
4. **Строгий TypeScript.** Включены `noUnusedLocals`/`noUnusedParameters` — мёртвый код роняет сборку.

## 🚀 Деплой
Прод: **https://syndicate-invest.ru** (PocketBase раздаёт SPA + API).

1. Закоммитить изменения и `git push` в `origin/main`.
2. Запустить деплой: локально `node deploy-ssh.js` **или** на сервере `cd /root/app && bash deploy.sh`
   (скрипт делает `git reset --hard origin/main`, `npm ci`, `npm run build`, копирует
   `dist` → `pb_public`, синхронит миграции PocketBase и перезапускает сервис).
